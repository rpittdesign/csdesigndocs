FORMS

Inputs, Selects, Textareas, Radio Buttons, Check Boxes

PREFACE

Inputs

Credit Card Input

1.0 STATIC
Interaction

  • Inherits standard input propertied and styles

2.0 CARD TYPE
Interaction

  • Inherits focus from inputs
  • Upon detecting card type logo of card type appears in input. Excess text flows underneath the logo.
  • Allows dashes and spaces. Excess characters are removed

Styles

  • label text - font-size: 20, #525252
  • input shadow - box-shadow: 0px 0px 3px 0px rgba(0,0,0,0.45)
  • input text - font-size 26, #333333

3.0 ERROR
Interaction

  • error triggers on blur and on key down and sometimes submit triggers error
  • Border,label, input text changes color
  • red invalid x appears next to input
  • red error message appears below input if needed pushes helper text down
  • animation - icon slides right, error message slides down
  • if error message size exceeds length of input it wraps
  • error always superscedes warnings

Styles

  • red: #D0021B
  • label becomes bold in addition to changing color
  • error message size 16px

Promo Code Input

1.0 HIDDEN
Interaction

  • Clicking or tapping link style button will hide text and reveal promo code box and button

Styles

  • Uses link button style on pattern library
  • Uses single pixel line above and below

2.0 VISIBLE
Interaction

  • Visible promo code and apply button

Styles

  • inherits inpt pattern and use link button style

3.0 APPLIED
Interaction

  • Applying a valid promo code triggers a [Code] successfully applied message.
  • Slides down similar to helper text

Styles

  • inherits helper text style and checkmark from validation

4.0 APPLIED EXPIRED
Interaction

  • Applying an expired promo code triggers a [Code] has expired message
  • Slides down similar to helper text

Styles

  • inherits helper text style and warning from validation

5.0 APPLIED INVALID
Interaction

  • Applying an invalid promo code triggers a [Code] applied is invalid message
  • Slides down similar to helper text

Styles

  • inherits helper text style and warning from validation

6.0 APPLYING A NEW CODE
Inherits

  • Adding any code beyond the first valid the user will be presented with helper text. Text warns them code will be replaced
  • Slides down similar to helper text

Styles

  • inherits helper text style

Selects

1.0 STATIC
Interaction

  • The select will scale to the content width - Selects are only used when there are between 6 to 15 options
  • Uses default dropdown menu to leverage best experience on touch devices - Dropdown menu should leverage categorization when possible

Styles

  • max-width: 500px - applies to 1024px width and larger - min-width: scales to 80% device width
  • height: 40px
  • border: 1px solid #525252
  • border-radius: 5px
  • background-color: #ffffff
  • placeholder text - Sentence case - label size - 18px
  • uses new triangle and left border

2.0 FOCUS
Interaction

  • Drop shadow will appear around active/focus input
  • Border darkens
  • This will be imperceptible on mobile devices with the select menu overlay

Styles

  • label text - font-size: 20, #525252
  • input shadow - box-shadow: 0px 0px 3px 0px rgba(0,0,0,0.45)
  • input text - font-size 26, #333333

3.0 VALID
Interaction

  • On exit if valid append check mark outside box only appears after fixing error
  • Error triggers on exit or button submit.
  • Exception being when a default is selected and user proceeds with default

Styles

  • checkmark scales according to device size. 24px

4.0 ERROR
Interaction

  • same function as input

Styles

  • red: #D0021B
  • label becomes bold in addition to changing color
  • error message size 16px

Textarea

1.0 STATIC
Interaction

  • Inherits input interaction and style
  • Height is suitable for multiple lines of text
  • Has character counter in bottom right. - Textarea does not use validation and error state

2.0 FOCUS
Interaction

  • Inherits focus form input
  • Character counter counts up as user types

3.0 WARNING
Interaction

  • Inherits warning from input
  • Character counter counts up as user types
  • Helper text - warning message slides down when user is within 10 characters of max.
  • Warning symbol slides down and slides counter left
  • Text wraps around counter

Delivery Date

1.0 DATE & TIME
Interaction

  • Default to Today and ASAP
  • Selecting date shows abbreviated month — Format Month Day, Year
  • Time uses 12 hour format rounds to nearest hour

Styles

  • Inherits style from selects
  • Icon for calendar and clock replace respective arrow

2.0 CALENDAR
Interaction

  • Inherits all from product egifting

Styles

  • Inherits from product egifting
  • Brands can configure highlight color and border color for hover

3.0 TIME
Interaction

  • Defaults to current hour and time AM/PM
  • Location + Timezone is a link to another module — defaults to purchaser closest location + timezone
  • Hour set up like a 12 hour clock widget
  • clicking outside module cancels time select
  • clicking done saves progress
  • if current hour of the same day is selected text defaults to ASAP

Styles

  • Time is large bold format uses dark grey
  • Am Pm function similar to a toggle. Highlighted one is bold and outlined in 3px border
  • Size of time and AM/PM scales down to account for smaller screen sizes
  • Location/TIMEZONE inherits style from links
  • highlighted hour inherits same color calendar uses
  • hovered hour inherits same border color calendar uses
  • tapping done saves new selected time
  • container style inherited from calendar

4.0 TIMEZONE
Interaction

  • Replaces clockpicker when Location/TIMEZONE is selected
  • Back arrow takes user to clock without saving
  • Defaults to current timezone of purchaser
  • Select has search function to filter through timezones
  • timezones listed by major city then timezone
  • done saves timezone and closes overlay

Styles

  • Inherits styles from corresponding elements
